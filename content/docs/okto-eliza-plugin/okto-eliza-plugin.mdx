## Okto Eliza Plugin Setup

Learn how to integrate the Okto Eliza Plugin with your existing Eliza project. This guide covers plugin configuration, authentication setup, and executing your first blockchain operations.
<br />

<Callout type="info" emoji="ðŸš€">
### Quick Start Example Available!
Skip the manual setup and get started in minutes with our <ExternalLink href="https://github.com/okto-hq/okto-sdk-eliza-agent-template">Example Repository</ExternalLink>. It includes:
- Pre-configured Okto Plugin integration
- Authentication setup
- Example AI agent implementation
</Callout>

<CollapsibleCallout type="warning" title="Important Setup Steps">
- Created your Okto Dashboard account
- Obtained your API keys from the dashboard
- Enabled the specific chains and tokens you plan to use in your application

Without completing these steps first, your application won't be able to interact with the Okto platform properly. Need help? Check out our <a href="/docs/developer-admin-dashboard" target="_blank" rel="noopener noreferrer">Dashboard Setup Guide</a> for detailed instructions.
</CollapsibleCallout>

<Steps>
<Step>
### Prerequisites

Before getting started, ensure you have the following:
- **Node.js (v23+) and pnpm:** <ExternalLink href="https://nodejs.org/">Download Node.js</ExternalLink>
- **Existing Eliza Project:** A working Eliza project where you want to add Web3 capabilities
- **Okto API Keys:** You need your `OKTO_CLIENT_PRIVATE_KEY` and `OKTO_CLIENT_SWA`. Obtain these from the Okto Dashboard.
- **Google OAuth Credentials:** Create OAuth 2.0 credentials in the <ExternalLink href="https://console.cloud.google.com/">Google Cloud Console</ExternalLink>.
- **LLM Provider Account (Optional):** Access to OpenAI, Anthropic, or another supported LLM provider.

</Step>

<Step>
### Installation

First, install the Okto Eliza Plugin package in your project:

```bash title="bash"
npm install @okto_web3/eliza-plugin
```
</Step>

<Step>
### Environment Setup

Configure your environment variables to connect with Okto services and authentication providers:

```bash title=".env"
# Okto Settings
OKTO_ENVIRONMENT=                # Optional. Defaults to "sandbox"
OKTO_CLIENT_PRIVATE_KEY=         # Required. Your client private key
OKTO_CLIENT_SWA=                 # Required. Your client SWA

# Google OAuth
GOOGLE_CLIENT_ID=                # Required
GOOGLE_CLIENT_SECRET=            # Required

# LLM Provider (Optional)
LLM_API_KEY=                     # Your LLM provider API key
```
</Step>

<Step>
### Google OAuth Configuration

Set up Google OAuth to enable user authentication:

1. Visit [Google Cloud Console](https://console.cloud.google.com)
2. Create a new project
3. Configure OAuth consent screen
4. Create OAuth 2.0 credentials
5. Add redirect URI: `http://localhost:5000`
6. Save credentials in environment variables
</Step>

<Step>
### Configuration

Configure the application settings and character parameters:

1. Edit `.env` with your credentials
2. Configure required environment variables
3. Review and update application settings as needed

<CollapsibleCallout type="tip" title="Customizing the LLM Provider">
Currently, this sample agent uses Groq as the LLM model provider. If you prefer a different provider:
1. Open `characters/okto.character.json`
2. Update the LLM provider configuration
3. Adjust character parameters as needed
</CollapsibleCallout>
</Step>

<Step>
### Basic Implementation

Initialize the Okto Plugin in your application:

```typescript
import {OktoPlugin} from "@okto_web3/eliza-plugin";

const oktoPlugin = new OktoPlugin()
return new AgentRuntime({
  databaseAdapter: db,
  token,
  modelProvider: character.modelProvider,
  plugins: [bootstrapPlugin, nodePlugin, oktoPlugin].filter(Boolean),
  services: [oktoPlugin.oktoService],
  // ... other configurations
});
```
</Step>

</Steps>

## Available Actions

The Okto Eliza Plugin provides the following actions:

| Action | Description | Github Source Code |
|---------|-------------|---------------|
| `OKTO_GET_PORTFOLIO` | Get user's token portfolio | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/getPortfolioAction.ts">Source Code</ExternalLink> |
| `OKTO_GET_ACCOUNT` | Get wallet accounts | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/getAccountAction.ts">Source Code</ExternalLink> |
| `OKTO_GET_CHAINS` | List supported networks | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/getChainAction.ts">Source Code</ExternalLink> |
| `OKTO_GET_NFT_COLLECTIONS` | Get NFT collections | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/getNftCollectionsAction.ts">Source Code</ExternalLink> |
| `OKTO_GET_ORDERS_HISTORY` | View transaction history | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/getOrdersHistoryAction.ts">Source Code</ExternalLink> |
| `OKTO_GET_PORTFOLIO_NFT` | Get NFT portfolio | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/getPortfolioNftAction.ts">Source Code</ExternalLink> |
| `OKTO_GET_TOKENS` | List supported tokens | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/getTokensAction.ts">Source Code</ExternalLink> |
| `OKTO_TRANSFER` | Transfer tokens | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/transferTokensAction.ts">Source Code</ExternalLink> |
| `OKTO_NFT_TRANSFER` | Transfer NFTs | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/nftTransferAction.ts">Source Code</ExternalLink> |
| `OKTO_SWAP` | Execute token swaps | <ExternalLink href="https://github.com/okto-hq/eliza-plugin/blob/main/src/actions/swapAction.ts">Source Code</ExternalLink> |

## Okto Eliza AI Agent Example

Learn how to run and customize our example implementation of the Okto Eliza Plugin.

<Steps>

<Step>
### Prerequisites

Before running the sample application, ensure you have:
- **Node.js:** v23+ (required) - <ExternalLink href="https://nodejs.org/">Download</ExternalLink>
- **Package Manager:** pnpm - <ExternalLink href="https://pnpm.io/installation">Install Guide</ExternalLink>
- **Okto Account:** Active account with API credentials
- **LLM Provider:** Access to OpenAI, Anthropic, or other supported providers
</Step>

<Step>
### Setup & Installation

Clone the repository and set up the initial project structure:

```bash title="bash"
# Clone the repository
git clone https://github.com/okto-hq/okto-sdk-eliza-agent-template

# Navigate and setup
cd okto-eliza-ai-agent-example
cp .env.sample .env

# Install dependencies
pnpm install
```
</Step>

<Step>
### Configuration

Configure the application settings and character parameters:

1. Edit `.env` with your credentials
2. Configure required environment variables
3. Review and update application settings as needed

Currently, this sample agent uses Groq as the LLM model provider. If you prefer a different provider:
1. Open `characters/okto.character.json`
2. Update the LLM provider configuration
3. Adjust character parameters as needed
</CollapsibleCallout>
</Step>

<Step>
### Launch

Start the application and begin testing:

```bash title="bash"
pnpm run start
```
