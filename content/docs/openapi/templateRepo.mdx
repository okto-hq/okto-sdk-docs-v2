## Getting Started

Find detailed instructions on how to set up the template app locally:

<Tabs items={["Node.js"]}>
  <Tab value="Node.js" >

1. Clone the repository:

```bash
git clone https://github.com/okto-hq/okto-sdkv2-ts-template.git
```

2. Change into the project directory and install dependencies:

```bash
cd okto-sdkv2-ts-template
npm install
```

3. Set up environment variables:

```bash
cp .env.sample .env
```

4. Add your API keys to the `.env` file:

```bash title=".env"
# The Okto environment "sandbox" or "production"
OKTO_ENVIRONMENT =

# Get the below values from Okto Developer Dashboard. Learn how here- https://docs.okto.tech/docs/developer-admin-dashboard
OKTO_CLIENT_PRIVATE_KEY =
OKTO_CLIENT_SWA =


#optional: only needed if using google authenticate
# For testing purposes, you can generate the id token from here
# - https://docs.okto.tech/docs/openapi/authenticate/google-oauth/get-token-id
# NOTE: The google ID token has a very short expiry. Please generate a new token just before running this code. You can check the expiry at jwt.io
GOOGLE_ID_TOKEN =


# when testing intents or using delegated access you will have to generate the okto auth token from session config
# For testing purposes, you can generate the okto auth token from our tool 'Session and Okto Auth Token Generator'
# - https://docs.okto.tech/tools
OKTO_AUTH_TOKEN =
```

  <CollapsibleCallout type="warning" title="Essential Setup Required">
    Before you begin, set up your [developer dashboard](https://dashboard.okto.tech) by making sure you have :
    - Created your [Okto Developer Dashboard](/docs/developer-admin-dashboard/dashboard-account) account
    - Get your [Okto credits](/docs/developer-admin-dashboard/okto-faucet) from the dashboard
    - Obtained your [API keys](/docs/developer-admin-dashboard/api-key) from the dashboard
    - Enabled the specific [Chains and Tokens](/docs/developer-admin-dashboard/wallet-control) you plan to use in your application
    - Optional: Enabled [Sponsorship](/docs/developer-admin-dashboard/sponsorship) for the desired chains

    If you need help, reach out to us on our <ExternalLink href="/docs/openapi/troubleshooting">troubleshooting form</ExternalLink> and we will contact you.
    </CollapsibleCallout>

  - Okto API Keys: You need your `CLIENT_PRIVATE_KEY` and `CLIENT_SWA`. Obtain these from the [Okto Developer Dashboard](https://dashboard.okto.tech/login).
  - Okto Account Setup: Ensure you have sufficient <ExternalLink href="https://dashboard.okto.tech/okto-credits">Okto Credits</ExternalLink>, enable the required <ExternalLink href="https://dashboard.okto.tech/wallet-controls">Chains and Tokens </ExternalLink>, and optionally <ExternalLink href="https://dashboard.okto.tech/paymaster">Activate Sponsorship</ExternalLink>.
  - Google OAuth Credentials (if using Google authentication): Create OAuth 2.0 credentials in the <ExternalLink href="/docs/openapi/authenticate/google-oauth/google-console-setup">Google Cloud Console</ExternalLink> to get your `GOOGLE_CLIENT_ID`.

5. Execute the script you want to run:

```bash
tsx path/to/your/script.ts
```
